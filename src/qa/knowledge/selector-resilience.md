# Resilient UI Selectors

## Principle
Target elements by their semantic role or explicit test identifiers, never by CSS class names, DOM structure, or brittle XPath expressions.

## Rationale
UI test fragility is the leading cause of test suite abandonment. When selectors are
tied to implementation details---CSS classes generated by a build tool, deeply nested
DOM paths, or styling-driven class names like `.btn-primary`---any visual redesign or
component refactor breaks dozens of tests that were functionally correct. The team
spends more time fixing selectors than catching bugs, and eventually stops trusting
or maintaining the suite.

Resilient selectors decouple tests from implementation. `data-testid` attributes are
the most reliable option: they survive CSS refactors, component library migrations,
and internationalization because they exist solely for testing. ARIA roles and
accessible names are the next best option, providing both testability and accessibility
verification. Playwright and Cypress both offer locator APIs that encourage resilient
patterns by default. The key insight is that selector strategy is a team decision, not
an individual one: developers must agree to add and maintain `data-testid` attributes,
and QA must agree to use them consistently.

## Pattern Examples

### 1. Selector Strategy Hierarchy (Best to Worst)

```
PREFER (most resilient)
  1. data-testid="login-submit-button"     -- Explicit test identifier.
  2. role="button" + name="Log in"         -- Semantic ARIA role + accessible name.
  3. label text: "Email address"           -- Form label association.
  4. placeholder text: "Enter email"       -- Visible text (may change with i18n).

AVOID (most brittle)
  5. CSS class: .btn-primary               -- Styling concern, changes often.
  6. Tag + index: div:nth-child(3)         -- Breaks when DOM order changes.
  7. XPath: //div[@class="form"]/div[2]/button  -- Structural coupling.
  8. Auto-generated: .css-1a2b3c           -- Build-tool hash, changes every build.
```

### 2. data-testid Best Practices

**Naming conventions:**

```html
<!-- Pattern: [component]-[element]-[qualifier] -->
<form data-testid="login-form">
  <input data-testid="login-email-input" type="email" />
  <input data-testid="login-password-input" type="password" />
  <button data-testid="login-submit-button" type="submit">Log in</button>
  <a data-testid="login-forgot-password-link" href="/forgot">Forgot password?</a>
</form>

<!-- Lists: use a stable identifier, not an index -->
<ul data-testid="product-list">
  <li data-testid="product-item-sku-001">Widget</li>
  <li data-testid="product-item-sku-002">Gadget</li>
</ul>

<!-- Dynamic content: embed a stable business identifier -->
<div data-testid="order-card-ORD-12345">
  <span data-testid="order-status-ORD-12345">Shipped</span>
</div>
```

**Strip test IDs from production builds (optional):**

```javascript
// babel.config.js
module.exports = {
  plugins: [
    // Remove data-testid attributes in production to keep the DOM clean.
    ...(process.env.NODE_ENV === "production"
      ? [["react-remove-properties", { properties: ["data-testid"] }]]
      : []),
  ],
};
```

### 3. Playwright Locator Strategies

```javascript
// tests/e2e/login.spec.js
const { test, expect } = require("@playwright/test");

test.describe("Login page", () => {

  test("logs in with valid credentials", async ({ page }) => {
    await page.goto("/login");

    // BEST: data-testid locator.
    await page.getByTestId("login-email-input").fill("user@example.com");
    await page.getByTestId("login-password-input").fill("SecurePass1!");
    await page.getByTestId("login-submit-button").click();

    await expect(page.getByTestId("dashboard-welcome")).toBeVisible();
  });

  test("shows validation errors (using ARIA roles)", async ({ page }) => {
    await page.goto("/login");

    // Submit without filling fields.
    await page.getByRole("button", { name: "Log in" }).click();

    // GOOD: ARIA role + accessible name.
    await expect(page.getByRole("alert")).toContainText("Email is required");
  });

  test("navigates via label text", async ({ page }) => {
    await page.goto("/login");

    // GOOD: Label association.
    await page.getByLabel("Email address").fill("user@example.com");
    await page.getByLabel("Password").fill("SecurePass1!");
    await page.getByRole("button", { name: "Log in" }).click();
  });

  test("displays product list items", async ({ page }) => {
    await page.goto("/products");

    // Locate a specific item by its test ID (which embeds the SKU).
    const product = page.getByTestId("product-item-sku-001");
    await expect(product).toBeVisible();
    await expect(product).toContainText("Widget");

    // Count items in the list.
    const items = page.getByTestId(/^product-item-/);
    await expect(items).toHaveCount(10);
  });
});
```

### 4. Cypress Locator Strategies

```javascript
// cypress/e2e/login.cy.js

describe("Login page", () => {

  it("logs in with valid credentials", () => {
    cy.visit("/login");

    // BEST: data-testid selector via custom command.
    cy.getByTestId("login-email-input").type("user@example.com");
    cy.getByTestId("login-password-input").type("SecurePass1!");
    cy.getByTestId("login-submit-button").click();

    cy.getByTestId("dashboard-welcome").should("be.visible");
  });

  it("validates required fields", () => {
    cy.visit("/login");

    cy.getByTestId("login-submit-button").click();

    // Check for ARIA role-based alert.
    cy.findByRole("alert").should("contain.text", "Email is required");
  });
});
```

**Custom Cypress command for data-testid:**

```javascript
// cypress/support/commands.js

/**
 * Custom command to select elements by data-testid.
 * Usage: cy.getByTestId("login-submit-button")
 */
Cypress.Commands.add("getByTestId", (testId) => {
  return cy.get(`[data-testid="${testId}"]`);
});

/**
 * Find within a parent by test ID.
 * Usage: cy.getByTestId("product-list").findByTestId("product-item-sku-001")
 */
Cypress.Commands.add(
  "findByTestId",
  { prevSubject: true },
  (subject, testId) => {
    return cy.wrap(subject).find(`[data-testid="${testId}"]`);
  }
);
```

### 5. Anti-Patterns with Corrections

```javascript
// BAD: CSS class selector - breaks when styling changes.
// page.locator(".btn-primary.large");
// GOOD:
page.getByTestId("checkout-submit-button");

// BAD: Nth-child selector - breaks when DOM order changes.
// page.locator("ul > li:nth-child(3)");
// GOOD:
page.getByTestId("product-item-sku-003");

// BAD: XPath with structural coupling - breaks on any wrapper div change.
// page.locator("//div[@class='container']/div[2]/form/button");
// GOOD:
page.getByRole("button", { name: "Submit order" });

// BAD: Auto-generated CSS module class - changes every build.
// page.locator(".LoginForm_submitButton__a1b2c");
// GOOD:
page.getByTestId("login-submit-button");

// BAD: Chained tag selectors - extremely fragile.
// page.locator("div > div > span > a");
// GOOD:
page.getByRole("link", { name: "Forgot password?" });

// BAD: Text content that changes with i18n.
// page.locator("text=Submit");  // English only.
// GOOD:
page.getByTestId("form-submit"); // Language-independent.

// BAD: ID selectors that are auto-generated by frameworks.
// page.locator("#react-select-3-option-0");
// GOOD:
page.getByTestId("country-select").getByRole("option", { name: "United States" });
```

### 6. Playwright Page Object with Resilient Selectors

```javascript
// tests/pages/checkout-page.js

class CheckoutPage {
  constructor(page) {
    this.page = page;

    // Define all selectors in one place.
    // If the HTML changes, only this class needs updating.
    this.shippingForm = page.getByTestId("checkout-shipping-form");
    this.firstName = page.getByLabel("First name");
    this.lastName = page.getByLabel("Last name");
    this.address = page.getByTestId("checkout-address-input");
    this.city = page.getByLabel("City");
    this.state = page.getByLabel("State");
    this.zip = page.getByLabel("ZIP code");
    this.continueButton = page.getByRole("button", { name: "Continue to payment" });
    this.orderSummary = page.getByTestId("checkout-order-summary");
    this.totalPrice = page.getByTestId("checkout-total-price");
    this.placeOrderButton = page.getByTestId("checkout-place-order");
    this.confirmationMessage = page.getByTestId("checkout-confirmation");
  }

  async goto() {
    await this.page.goto("/checkout");
  }

  async fillShippingAddress({ first, last, address, city, state, zip }) {
    await this.firstName.fill(first);
    await this.lastName.fill(last);
    await this.address.fill(address);
    await this.city.fill(city);
    await this.state.selectOption(state);
    await this.zip.fill(zip);
  }

  async continueToPayment() {
    await this.continueButton.click();
  }

  async placeOrder() {
    await this.placeOrderButton.click();
    await this.confirmationMessage.waitFor({ state: "visible" });
  }

  async getTotal() {
    return this.totalPrice.textContent();
  }
}

module.exports = { CheckoutPage };
```

### 7. Selector Audit Script

```javascript
// scripts/audit-selectors.js
// Scan test files for fragile selector patterns and report violations.

const fs = require("fs");
const path = require("path");
const glob = require("glob");

const FRAGILE_PATTERNS = [
  {
    regex: /\.locator\(["']\.[\w-]+["']\)/g,
    message: "CSS class selector",
    severity: "warning",
  },
  {
    regex: /\.locator\(["']#[\w-]+["']\)/g,
    message: "ID selector (verify it is stable, not auto-generated)",
    severity: "info",
  },
  {
    regex: /:nth-child\(\d+\)/g,
    message: "nth-child selector",
    severity: "error",
  },
  {
    regex: /\/\/[\w[\]@='".*]+/g,
    message: "XPath expression",
    severity: "error",
  },
  {
    regex: /\.css-[\w]+/g,
    message: "Auto-generated CSS module class",
    severity: "error",
  },
];

function auditFile(filePath) {
  const content = fs.readFileSync(filePath, "utf-8");
  const lines = content.split("\n");
  const violations = [];

  lines.forEach((line, index) => {
    for (const pattern of FRAGILE_PATTERNS) {
      const matches = line.match(pattern.regex);
      if (matches) {
        violations.push({
          file: filePath,
          line: index + 1,
          severity: pattern.severity,
          message: pattern.message,
          match: matches[0],
        });
      }
    }
  });

  return violations;
}

// Scan all test files.
const testFiles = glob.sync("tests/**/*.{spec,test}.{js,ts}");
let totalViolations = 0;

for (const file of testFiles) {
  const violations = auditFile(file);
  for (const v of violations) {
    console.log(
      `[${v.severity.toUpperCase()}] ${v.file}:${v.line} - ${v.message}: ${v.match}`
    );
    totalViolations++;
  }
}

console.log(`\nTotal violations: ${totalViolations}`);
process.exit(totalViolations > 0 ? 1 : 0);
```
