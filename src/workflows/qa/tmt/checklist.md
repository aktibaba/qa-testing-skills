# Teaching Session Quality Checklist

Use this checklist to validate learning sessions and track progress.

---

## 1. Assessment Quality

- [ ] Learner's current level has been accurately assessed
- [ ] Assessment covers both theoretical knowledge and practical skills
- [ ] Technology stack and preferences are identified
- [ ] Learning goals are documented
- [ ] Recommended track is appropriate for the learner's level

## 2. Curriculum Fit

- [ ] Curriculum matches the assessed level (not too easy, not too hard)
- [ ] Topics build progressively (prerequisites before advanced topics)
- [ ] Curriculum uses the learner's technology stack for examples
- [ ] Session count is realistic for the scope of material
- [ ] Curriculum can be adjusted based on learner feedback

## 3. Session Delivery

- [ ] Each session follows the structured format (warm-up, concept, example, exercise, review, summary)
- [ ] Explanations use concrete examples, not just theory
- [ ] Code examples are complete and runnable
- [ ] Exercises have clear instructions and expected outcomes
- [ ] Difficulty matches the learner's level
- [ ] Session stays within the time budget (`{session_duration}`)

## 4. Exercises and Practice

- [ ] Every session includes at least one hands-on exercise
- [ ] Exercises reinforce the session's key concept
- [ ] Starter code is provided for beginners
- [ ] Multiple difficulty levels available (basic, challenge, bonus)
- [ ] Solutions are available for self-check
- [ ] Exercises use the learner's actual codebase when possible

## 5. Progress Tracking

- [ ] Session completion is logged with date and topics covered
- [ ] Key concepts mastered are recorded
- [ ] Concepts needing review are flagged
- [ ] Exercises completed and scores are tracked
- [ ] Next session recommendations are personalized
- [ ] Progress can be reviewed by the learner at any time

## 6. Knowledge Retention

- [ ] Each session begins with a recall question from previous material
- [ ] Spaced repetition is applied for critical concepts
- [ ] Knowledge check exercises test understanding, not just recall
- [ ] Learner can explain concepts in their own words
- [ ] Learner can apply concepts to new (unseen) scenarios

---

## Learner Progress Levels

| Level | Description | Evidence |
|-------|-------------|----------|
| Novice | Can follow guided examples | Completes exercises with hints |
| Practitioner | Can write tests independently | Writes tests without guidance |
| Proficient | Can design test strategies | Creates test plans and architectures |
| Expert | Can mentor others | Teaches concepts and reviews test code |
